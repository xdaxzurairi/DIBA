# 📊 Diagram Entry Format
*Template for ASCII diagram entries*

## Required Fields
- Diagram name and description
- The ASCII diagram
- Explanation
- Key points

## Template

```markdown
# [Diagram Name]
*[What this diagram shows]*

## Diagram

```
[ASCII diagram here]

Use box-drawing characters:
┌ ┐ └ ┘ │ ─ ├ ┤ ┬ ┴ ┼
Arrows: → ← ↑ ↓ ▲ ▼ ◀ ▶
```

## Explanation
[2-3 sentences explaining what this diagram represents and how to read it]

## Key Points
- [Important point 1]
- [Important point 2]
- [Important point 3]

## Components

| Component | Purpose |
|-----------|---------|
| [Box 1] | [What it represents] |
| [Box 2] | [What it represents] |
| [Arrow] | [What the flow means] |

## Variations (if applicable)
[Alternative versions or scenarios]

## Related
- [Related diagram]
- [Related architecture entry]

---
*Documented: [Date]*
```

## Example Entry Names
- `order-flow` - Order processing flow
- `booking-flow` - Booking confirmation flow
- `mobile-layout` - Mobile app screen structure
- `auth-flow` - Authentication flow
- `data-migration` - Data migration process

## ASCII Diagram Tips
- Use consistent box sizes
- Align arrows properly
- Add labels inside or beside boxes
- Keep it readable (not too complex)
