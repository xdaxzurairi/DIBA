## Perbincangan Design — Sistem Cabutan Tapak Konvo

Sesi real-time discussion untuk reka bentuk workflow dan DB schema:

### Workflow (6 Fasa — dikunci)
- FASA 1: Admin cipta Sesi Cabutan
- FASA 2: Penyewa Konvo daftar & isi borang
- FASA 3: Admin semak → lulus | gagal → email
- FASA 4: Admin buka kehadiran → tandakan Hadir
- FASA 5: Admin jalankan cabutan manual → PDF auto + email, tempoh tawaran 48 jam
- FASA 6: Cascade senarai tunggu 1 kali sahaja

### Keputusan Design
- `status_permohonan`: `pending | lulus | gagal` (lulus_bersyarat dibuang)
- Cabutan: admin pilih manual (bukan random algorithm)
- Surat keputusan: PDF auto-generate
- Cascade senarai tunggu: 1 kali sahaja
- `status_pengurusan`: tangguh, nilai belum confirm
- Borang tambah field **julat harga jualan** — dua input bebas: `harga_jualan_min` + `harga_jualan_max` (DECIMAL 10,2)
