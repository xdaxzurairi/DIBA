## Sesi Tengahari — ea_newv3 SQL Tuning (10:05–10:25)

### Objective
Audit dan tuning semua SQL queries dalam sistem eAduan v3 (PHP + MSSQL) supaya laju dan konsisten.

### Audit Dapatan
DIBA scan 25 fail PHP yang mengandungi SQL — 5 isu dikenal pasti:

| Tahap | Fail | Isu |
|-------|------|-----|
| Kritikal | `semak.php` | N+1 queries dalam foreach loop (20 extra queries per page) |
| Sederhana | `borang.php` | Query `SELECT ... FROM em` duplikat (2× sama) |
| Sederhana | `simpan.php` | Zone lookup 2 round trips → boleh jadi 1 JOIN |
| Sederhana | `detail.php` | 10+ sequential single-field lookups |
| Minor | `get_seksyen.php` | Implicit comma JOIN (bukan explicit INNER JOIN) |

### Fix Dilaksanakan

**`semak.php` — Batch lookup (N+1 → 2 queries)**
- Kumpul semua `bl_id` unik → satu `WHERE bl_id IN (...)` query
- Kumpul semua `(section, prob_group, prob_type)` combo → satu batch OR query
- Loop guna PHP map lookup sahaja — tiada lagi DB query dalam foreach

**`borang.php` — Buang query duplikat**
- Line 137 `SELECT bl_id, fl_id, rm_id FROM em WHERE em_number = ?` → ganti dengan `$space_row = $row`
- `$row` sedia ada hasil dari query em di line 115 (mengandungi kolum yang sama)

**`simpan.php` — Zone lookup 1 JOIN**
- 2 queries (get bl → check zone_uitm) → `INNER JOIN afm.zone_uitm z ON z.zone_id = b.zone_id`
- Kalau zone_id tidak wujud dalam zone_uitm, JOIN return null secara semula jadi

**`detail.php` — 2 consolidated JOIN queries**
- 8 sequential lokasi queries → 1 JOIN query (state + site + bl + bldg + fl + rm sekaligus)
- 3 sequential probtype queries → 1 JOIN query (section + prob_group + probtype sekaligus)
- Infra & subinfra kekal berasingan (conditional logic)

**`get_seksyen.php` — Explicit INNER JOIN**
- `FROM section a, probtype b WHERE a.section = b.section` → `INNER JOIN probtype b ON b.section = a.section`

### Commit
- `2881727` — `perf: SQL tuning - batch queries, eliminate N+1, consolidate lookups`
- 5 fail, +100/-65 lines
